# Fix Overlay Lag - Disable Hardware-Accelerated GPU Scheduling (HAGS)

If you're experiencing noticeable overlay lag (high D3D_Present times) even when CS2 is running at high FPS, disabling Hardware-Accelerated GPU Scheduling (HAGS) can significantly improve overlay performance.

<Steps>
  <Step title="Disable Hardware-accelerated GPU Scheduling">
    1. Press **Win + I** to open **Windows Settings**
    2. Go to **System → Display**
    3. Scroll down and click **Graphics settings**
    4. Toggle **OFF** **Hardware-accelerated GPU scheduling**
    5. **Restart your computer** (this change requires a full reboot)
  </Step>
</Steps>

After restarting, launch CS2 + NinjaHackz. The overlay should feel significantly smoother and more responsive.

<Note>
  Disabling HAGS is currently the most reliable fix for external overlay performance.
</Note>

## How to Re-enable HAGS (if needed)

Follow the exact same steps above and toggle the setting **back ON**, then restart your PC again.

## Why This Fixes the Lag

Hardware-Accelerated GPU Scheduling (HAGS) lets the GPU manage its own command queues and scheduling directly. While this is beneficial for many modern games, it creates problems for external overlays like NinjaHackz.

NinjaHackz works by injecting into the game’s render pipeline and using DComp (Desktop Composition) to draw on top. With HAGS enabled, the GPU’s hardware scheduler has trouble efficiently interleaving the game’s commands with the overlay’s extra draw calls. This causes presentation delays, stuttering, and high `D3D_Present` times in the overlay — even when the game itself runs smoothly.

Disabling HAGS forces Windows to fall back to the legacy CPU-based scheduler. This method is much more compatible with the kind of mixed workload created by injected overlays, resulting in smoother frame delivery and lower overlay latency.
